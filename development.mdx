## Multi-Tenancy

Workflow supports multi-tenant (company) and multi-organization (branch) architecture with Row-Level Security enforced at the database level. Each user belongs to one company and can access one or more organizations within that company.

<Accordion title="ภาษาไทย">
Workflow รองรับสถาปัตยกรรมแบบ multi-tenant (บริษัท) และ multi-organization (สาขา) โดยใช้ Row-Level Security ในระดับฐานข้อมูล ผู้ใช้แต่ละคนสังกัดบริษัทหนึ่งและสามารถเข้าถึงได้หนึ่งหรือหลายองค์กรในบริษัทนั้น
</Accordion>

## Roles

Users can hold **multiple roles simultaneously** — permissions are merged as a union across all assigned roles.

*ผู้ใช้สามารถมีได้**หลายบทบาทพร้อมกัน** สิทธิ์จะถูกรวมกันแบบ union จากทุกบทบาทที่ได้รับมอบหมาย*

| Role | Description |
|------|-------------|
| **Admin** | Full access to all modules and settings |
| **Manager** | Approvals, reporting, all operational modules |
| **Procurement** | PRs, POs, Vendors, Goods Receipts |
| **Warehouse** | GRs, Inventory, Stock Adjustments, Delivery Notes |
| **Production** | Work Orders, BOMs, Inventory |
| **Engineer** | BOMs, Products, Job Controls |
| **Viewer** | Read-only access to all modules |

## Assigning Roles

1. Go to **Settings** → **Users**
2. Click **Invite User** or select an existing user
3. Toggle one or more roles
4. Save — changes take effect immediately

*ไปที่ **Settings** → **Users** แล้วเลือกผู้ใช้หรือกด **Invite User** จากนั้นเปิด/ปิดบทบาทที่ต้องการ*
